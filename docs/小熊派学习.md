# 避坑指南：小熊派 BearPi-HM Nano 开发环境搭建与编译全流程

最近在搭建小熊派 BearPi-HM Nano 的 OpenHarmony 开发环境时，踩了不少坑，尤其是 `hb` 编译时报错 `Please call hb utilities inside source root directory`，折腾了半天发现是 `ohos-build` 版本不匹配导致的。
下面是验证可行的完整流程，帮助大家一步到位，避开所有雷区！

> [!TIP]
> - 确保你已经安装了 `Python 3.10` 并配置了环境变量。
> - 这里使用粤嵌配置好环境的虚拟机，版本为 `Ubuntu 20.04`，无资源请跳转我的主页进行获取
## 第一部分：编译环境搭建

### 一、基础环境准备

#### 1. 更新系统并安装 VMware Tools (可选，针对虚拟机用户)

如果你使用的是 VMware 虚拟机，建议先安装 VMware Tools 以支持文件共享和复制粘贴。

```bash
cd ~
# 假设你的 VMware Tools 安装包在虚拟光驱中，将其移动到用户目录(注意路径中的空格需要转义以及末尾的 .为当前目录)
mv /media/gec/VMware\ Tools/VMwareTools-10.3.23-16594550.tar.gz .

# 使用tar解压安装包
tar -xzf VMwareTools-10.3.23-16594550.tar.gz

# 进入解压后的目录
cd VMwareTools-10.3.23-16594550

# 安装 VMware Tools
sudo ./vmware-install.pl # 一路按 Enter 键即可

# 更新系统
sudo apt update

# 安装 VMware Tools 相关软件包，同样一路按 Enter 键即可
sudo apt install open-vm-tools open-vm-tools-desktop -y

# 重启系统
sudo reboot
```

#### 2. 安装 Node.js 和 hpm CLI (可选)

OpenHarmony 的包管理工具 `hpm` 依赖于 Node.js。如果你没有使用已配置完成的系统请先执行：

```bash
# 安装 Node.js (以 Node.js 14 为例，兼容性较好)
curl -sL https://deb.nodesource.com/setup_14.x | sudo -E bash - && sudo apt install -y nodejs

# 安装 hpm 命令行工具
sudo npm install -g @ohos/hpm-cli
```

### 获取源码与依赖

#### 1. 解压源码包

下载源码压缩包，点击跳转：[下载链接](https://repo.harmonyos.com/#/cn/solution/@bearpi%2Fbearpi_hm_nano_agriculture)，假设你将源码包放在了 VMware 的共享文件夹中。（共享文件夹如何开启，详见：https://blog.csdn.net/Cixil/article/details/105659835）

```bash
cd ~
# 创建 ohos 目录，我们使用该目录管理所有 OpenHarmony 项目
mkdir ohos

cd ohos

mv /mnt/hgfs/iteaching/my_bearpi_hm_nano_agriculture.zip .

# 解压源码包
unzip my_bearpi_hm_nano_agriculture.zip

# 进入解压后的目录
cd my_bearpi_hm_nano_agriculture
```

#### 2. 初始化并拉取依赖

使用 `hpm` 工具下载该开发板所需的所有依赖包。

```bash
# 初始化项目
hpm init -t dist
# 拉取依赖包
hpm i @bearpi/bearpi_hm_nano_agriculture
```

_(注：此过程因网络原因可能较慢，请耐心等待)_

### 三、安装编译工具链

> [!IMPORTANT]
> 千万不要直接执行 `python3 -m pip install --user ohos-build`！
> 默认安装的最新版 (`0.4.6` 及以上) 与小熊派当前的开源代码结构不匹配，会导致后续编译时 `hb` 找不到目录而报错。

```bash
python3 -m pip install --user ohos-build==0.4.3
```

#### 补充：配置环境变量

`ohos-build` 安装后，`hb` 可执行文件通常位于 `~/.local/bin/` 目录下。为了确保编译时能找到它，建议将其加入环境变量：

```bash
# 添加环境变量到 ~/.bashrc 中
echo 'export PATH=~/.local/bin:$PATH' >> ~/.bashrc

# 刷新环境变量
source ~/.bashrc
```

### 四、一键编译

完成以上步骤后，直接运行编译命令：

```bash
hpm dist
```
*看到终端滚动编译日志，并最终输出绿色字体的 `success`，就说明固件编译成功了！生成的固件在 `out/` 目录下，可以直接使用烧录工具烧录到开发板中。*

### 五、配置 Samba 实现双向文件互传

在实际开发中，我们经常需要用 Windows 下的编辑器（如 VS Code）修改代码，或者用 Windows 下的烧录工具烧录固件。来回用 U 盘或命令拷贝太麻烦。为了快速便捷开发，提供如下方案：

<!-- tabs:start -->

#### **方案1**

安装open-vm-tools-desktop，通过虚拟机文件夹图形化拖拽文件。

#### **方案2**

将out内的编译文件复制进入共享文件夹。

```bash
cp -r out/* /mnt/hgfs/iteaching/
```

#### **方案3**

使用 SecureCRT(支持更多协议) 或 MobaXterm Pro (推荐，易上手) 连接 Ubuntu 虚拟机，直接在 Windows 下操作 Ubuntu 文件系统。

#### **方案4**

配置 Samba 实现双向文件互传。

<!-- tabs:end -->

我主要讲一下通过配置 **Samba**，让 Ubuntu 的 `ohos` 目录直接作为网络驱动器映射到 Windows 下，像本地磁盘一样访问！

#### 1. 在 Ubuntu 中安装和配置 Samba

在终端依次执行以下命令：

```bash
# 1. 安装 Samba
sudo apt install samba -y

# 2. 备份原配置文件
sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bak

# 3. 编辑配置文件
sudo nano /etc/samba/smb.conf
```

在打开的配置文件**最末尾**，添加以下内容（注意 `path` 改为你实际的源码路径）：

```ini
[ohos]
   comment = OpenHarmony Source
   path = /home/gec/ohos
   browseable = yes
   read only = no
   create mask = 0777
   directory mask = 0777
   valid users = gec
```

按 `Ctrl+O` 保存，`Enter` 确认，`Ctrl+X` 退出。
接着设置 Samba 访问密码（建议和当前 Ubuntu 登录密码保持一致，避免混淆）：

```bash
# 为当前用户 gec 设置 Samba 密码（会提示输入两次密码）
sudo smbpasswd -a gec

# 重启 Samba 服务生效
sudo service smbd restart
```

#### 2. 在 Windows 中映射网络驱动器

1. 在 Ubuntu 终端输入 `ip a`，查看虚拟机的 IP 地址（通常是 `192.168.x.x`）。
2. 回到 Windows 桌面，按 `Win + R` 打开运行窗口。
3. 输入 `\\` 加上 Ubuntu 的 IP 地址（例如 `\\192.168.1.100`），回车。
4. 弹出认证窗口，输入用户名 `gec` 和你刚才设置的 Samba 密码。
5. 此时你可以看到 `ohos` 文件夹，右键点击它 -> **映射网络驱动器** -> 选择一个盘符（如 `Z:`） -> 完成。
   现在，你的 Windows 资源管理器里就多出了一个 `Z:` 盘，它就是 Ubuntu 里的 `~/ohos` 目录！

> [!WARNING]
> 虽然映射网络驱动器非常方便，但是跨系统的文件系统不支持 Linux 的软链接和权限

**正确的开发流程**：

- **在 Windows (Z: 盘) 下**：用 VS Code 阅读、修改代码；运行烧录工具烧录 `.bin` 固件。
- **在 Ubuntu 本地终端下**：执行 `cd ~/ohos/my_bearpi_hm_nano_agriculture` 和 `hpm dist` 编译代码。
- **绝对不要**：在 Windows 下直接运行编译脚本，或者在 Ubuntu 里 `cd /mnt/...` 到 Windows 盘符下执行编译。

---

### 问题总结



+ 执行 `hpm dist` 报错 `Exception: Please call hb utilities inside source root directory` 怎么办？ +

你安装了高版本的 `ohos-build`。请执行 `python3 -m pip uninstall ohos-build -y` 卸载，然后重新安装指定版本 `python3 -m pip install --user ohos-build==0.4.3`。
